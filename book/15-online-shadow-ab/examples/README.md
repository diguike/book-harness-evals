# 第 15 章配套：线上持续评估管道

一套不依赖真实 LLM、确定性可复现的线上持续评估闭环，复现正文开头那次"离线全绿、线上慢慢烂掉"的故障：新版把升级判断从模型自判改成关键词表后，大量本该自助的常规扩容操作被误升级——这类样本离线任务集里没有，只有跟到线上才测得出来。

## 跑起来

```bash
npm install
npm run demo       # 跑整条闭环：影子 → A/B → 灰度 → 信号回流
npm run typecheck  # 类型检查
```

## 文件导览

| 文件 | 作用 |
|---|---|
| `src/adapter.ts` | 全书 canonical 的 `HarnessAdapter` 接口（第 5 章定义），与全书逐字一致 |
| `src/stats.ts` | 统计件，与第 4 章同形：Wilson 区间 / 双比例 z 检验 / Bonferroni / 样本量估计 |
| `src/proxy-signals.ts` | 跨线上线下**对齐口径**的代理信号判定（复用第 7 章状态判定形状）——线上没 oracle，用硬信号近似 |
| `src/mock-harness.ts` | 值班助手桩：老版（模型自判升级）vs 新版（关键词表，过度升级）+ 真实流量生成器 |
| `src/shadow.ts` | 影子比对器：候选版只读 dry-run，把流量分到 一致/更优/更差/新增违规 四桶 + 影子门禁 |
| `src/ab.ts` | A/B 分流：稳定哈希分桶 + 各臂通过率 + Wilson CI + 显著性判定 |
| `src/canary.ts` | 灰度放量：阶梯放大 + 守护性门限（一票否决）+ 趋势性门限（看显著性）+ 自动回滚 |
| `src/run.ts` | 串起全流程，并演示把"候选更差"样本回流成带 oracle 的离线 `EvalTask` |

## 你会看到什么

- **影子**：四桶分布里"候选更差"堆起一大批常规操作流量，影子门禁直接拦截——这次故障本可在上线前挡住。
- **A/B**：新版通过率带 Wilson 区间，和老版的差异做双比例 z 检验、Bonferroni 校正，给出"是否显著变差"。
- **灰度**：候选版从 1% 起阶梯放量，守护性 / 趋势性门限任一触线就回滚停在上一安全档。
- **回流**：把"候选更差"样本规整成带 `oracle` 的 `EvalTask`，交人工审一道再入回归集（不做无人值守自动入集）。

把 `src/mock-harness.ts` 里的 `MockOncallAdapter` 换成第 5 章的 `MastraOncallAdapter`，这套管道就能直接对接真实 Mastra 值班助手与 AI Tracing 落库的线上数据。
